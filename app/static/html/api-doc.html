Accueil

Bienvenue dans la documentation de l'API du projet de session INF5190. Cette API vous permettra d'obtenir une foule d'infos sur les installations de la Ville de Montréal.

/installations

/installations get

get

Retourne la liste des installations

get /installations

Retourne la liste des installations

  • Request
  • Response

Query Parameters

  • arrondissement: (string)

    Une chaìne de caractères qui représente le nom de l'arrondissement

    Example:

    Verdun

HTTP status code 200

La liste des installations a été correctement envoyée.

Body

Media type: application/json

Possible types:

  • Glissade

    • id: required(number)
    • nom: required(string)
    • arrondissement: required(object)
      • id: required(number)
      • nom: required(string)
      • cle: (string)
      • date_maj: (datetime-only)
    • ouvert: (boolean)
    • deblaye: (boolean)
    • condition: (string)

  • InstallationAquatique

    • id: required(number)
    • nom: required(string)
    • arrondissement: required(object)
      • id: required(number)
      • nom: required(string)
      • cle: (string)
      • date_maj: (datetime-only)
    • type: (string)
    • adresse: (string)
    • propriete: (string)
    • gestion: (string)
    • equipement: (string)

  • Patinoire

    • id: required(number)
    • nom: (string)
    • arrondissement: required(object)
      • id: required(number)
      • nom: required(string)

/abonnement

/abonnement post

post

S'abonner aux alertes par courriel

post /abonnement

S'abonner aux alertes par courriel

  • Request
  • Response

Query Parameters

  • subscriber: required(json)

    Les informations de la personne

    Example:

    {
  "full_name":"Alex",
  "email":"courriel@hebergeur.com",
  "boroughs_to_follow":"[1,2,3]"
}

HTTP status code 201

La personne s'est abonné avec succès.

Body

Media type: application/json

Type: json

Content:

{
 "$schema" : "https://json-schema.org/draft/2020-12/schema#",
 "title" : "Nouvel abonné",
 "description" : "Valider les informations d'un nouvel abonné",
 "type" : "object",
 "properties" : {
    "full_name" : {
       "description" : "Nom complet",
       "type" : "string",
       "minLength" : 1,
       "maxLength": 64
    },
    "email" : {
       "description" : "Courriel",
       "type" : "string",
       "minLength" : 1,
       "maxLength" : 64,
       "pattern" : "\\b[A-Za-z0-9._%+-]+@[A-Za-z0-9.-]+\\.[A-Z|a-z]{2,}\\b"
    },
    "boroughs_to_follow" : {
       "description" : "ID des arrondissements à suivre",
       "type" : "array",
       "items" : {
          "type" : "integer"
       },
       "maxLength" : 255,
       "uniqueItems" : true
    }
 },
 "required" : [
    "full_name",
    "email"
 ]
}

Example:

{
  "full_name":"Alex",
  "email":"courriel@hebergeur.com",
  "boroughs_to_follow":"[1,2,3]"
}

HTTP status code 400

Les données fournies ne sont pas valides.

Body

Media type: application/json

Type: object

Properties
  • status: required(string)
  • message: required(string)

Example:

{
  "error": "Les données fournies ne sont pas valides."
}
